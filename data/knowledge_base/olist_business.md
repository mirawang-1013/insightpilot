# Olist 业务背景知识

本文档解释 Olist 数据集的业务上下文和数据特点。
LLM 在做业务分析或给建议时需要这些背景知识。

---

## Olist 是什么

**定义**：Olist 是巴西最大的电商市场聚合平台，类似中国的"有赞"或美国的"Shopify + Marketplace"。

**业务模式**：
- 卖家（sellers）通过 Olist 把商品上架到主流巴西电商平台（Mercado Livre、Magalu 等）
- 用户（customers）在前端电商下单
- Olist 处理订单、物流、支付分账

**数据集时间跨度**：2016 年 9 月 - 2018 年 10 月

**总规模**：约 99,441 条订单 / 96,096 个独立用户 / 3,095 个卖家 / 73 个商品品类

---

## 巴西市场特殊性

**地理因素**：
- 巴西国土面积大，南北向跨度长，物流成本是关键考量
- SP（圣保罗）、RJ（里约）、MG（米纳斯吉拉斯）三州合计占订单约 65%
- 北部和东北部州（AM、PA、MA）配送时效慢、订单量少

**支付偏好**：
- credit_card（信用卡）最常用，约 75% 订单
- boleto（巴西特有的银行付款单）约 19%，下单到付款有滞后
- voucher（优惠券）和 debit_card（借记卡）占比小
- **分期付款是巴西消费习惯** —— 信用卡平均分 3-4 期

**评论文化**：
- 巴西用户给评分较慷慨，平均评分 ~4.1/5
- < 3 分通常是物流问题（晚到、破损）

## 订单状态枚举

`order_status` 字段的值及含义：

| 状态 | 含义 | 在分析中怎么处理 |
|---|---|---|
| `delivered` | 已成功送达 | **营收/评分分析的核心** |
| `shipped` | 已发货未送达 | 算 GMV 时通常算上 |
| `canceled` | 已取消 | **必须过滤掉** |
| `unavailable` | 商品缺货 | 通常过滤 |
| `invoiced` | 已开发票未发货 | 中间状态，少见 |
| `processing` | 处理中 | 中间状态 |
| `created` | 刚创建 | 中间状态 |
| `approved` | 付款通过未发货 | 中间状态 |

**默认建议**：营收分析用 `IN ('delivered', 'shipped')`，订单数分析视情况包含中间状态。

## 数据集已知的局限

**1. 没有用户画像**
- 没有年龄、性别、注册时间字段
- 用户行为分析只能从订单 / 评论反推

**2. 没有商品库存数据**
- 不能分析"缺货影响"

**3. 没有营销/广告数据**
- 没法直接算 ROAS / CPC / 转化漏斗
- "投资建议"类问题只能基于销售数据反推

**4. 时间不连续**
- 2016 年订单很少（Olist 早期）
- 2018-09 后数据缺失（数据集导出截止）
- 趋势分析尽量用 2017 全年（最完整）

**5. 评论字段是葡语**
- `order_reviews.review_comment_message` 是葡语原文
- 分析评论情感需要先翻译

## 常见的业务洞察模板

### "投资建议" 类问题怎么回答

如果用户问"该投资哪个品类/州/卖家"，分析框架应包含：

1. **市场规模** —— 总营收、订单数
2. **市场增速** —— 同比 / 环比变化
3. **盈利质量** —— 客单价、复购率
4. **风险信号** —— 评分、配送延迟、退款率
5. **竞争格局** —— 卖家集中度

**输出格式建议**：
```
品类 X：
  - 规模：年营收 XX 万雷亚尔（行业第 N）
  - 增速：YoY +XX%
  - 客单价：XX BRL（高于 / 低于平均）
  - 评分：4.X / 5（用户体验 OK / 有风险）
  - 集中度：Top 10 卖家占 XX%（高 / 中 / 低）
  - 建议：✓ 优先 / ⚠️ 观察 / ✗ 慎入
```

### "对比 Top 5" 类问题

要求：
1. 用排名表（不是单一指标），让对比维度清晰
2. **正负面信号都给** —— 不要只说好不说坏
3. 给出"哪个最值得"的明确结论

### "分析 X 和 Y 的关系" 类问题

应该：
1. 先取相关数据
2. 算相关系数（pandas `df.corr()`）
3. 画散点图看趋势
4. 区分"相关"和"因果"，不要直接说"X 导致 Y"

## "敏感结论" 的判定（第五阶段会用）

以下结论属于**敏感**，建议触发人工审批：

- "建议投资 X 万雷亚尔到 Y 品类"（具体金额建议）
- "应该裁撤 X 品类 / X 卖家"（业务决策）
- "X 品类用户体验差"（声誉相关）
- 涉及 GDPR / LGPD（巴西数据保护法）的用户隐私分析

**安全的结论**（不需要审批）：
- 趋势描述（"营收上升了 X%"）
- 数据汇总（"Top 5 品类是 ..."）
- 中性洞察（"配送延迟和评分负相关"）
